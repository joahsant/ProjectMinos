# Feature Intake Workflow

## Purpose
- Define the repeatable intake workflow for new features and major feature expansions.

## Core Intake Checks
- objective
- primary surface
- user flow
- data source
- refresh policy
- battery impact
- failure states
- future-scope boundaries
- validation path

## Output
- product premises
- open decisions
- safe autonomous assumptions
- validation path
- whether Product, UX/UI, Market Data, Engineer, QA, or Historian must participate

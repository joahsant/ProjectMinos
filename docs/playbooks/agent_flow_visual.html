Project Minos multiagent runtime

Agent Flow Map

This page translates the repository governance and Python orchestration into a single visual flow. The key idea is simple: every request enters through Lead, gets classified, opens a gate sequence, activates roles at the required depth, routes to the next owner, and optionally persists request state plus role reports under the local runtime directory.

Lead intake Product premise work UX/UI validation Market-data review Implementation route Validation route Truth consolidation

Flow canvas

1. Intake
Entry

User request

Every request starts here and must enter through Lead / Orchestrator. No direct Engineer-first path is allowed.

Lead

Interpretation + classification

Lead makes the intake explicit: interpretation, classification, gate state, activated roles, depth, and next owner.

Routing signal

RequestSignals payload

Signals such as UX impact, new states, ambiguity, structural change, and polling or data changes drive the route.

2. Decision layer
Classifier

Change class

One of: trivial patch, localized change, feature flow, or structural initiative.

Mode

Task mode

One of: advisory, review, planning, or implementation. Classification and mode stay separate.

Output

Lead report

The orchestrator emits assumptions, risks, gate sequence, role activations, and a recommendation for the next role.

3. Role activation
Role stack

Product Strategist

Activated deeply when behavior, states, retries, or scope boundaries are unresolved.

Role stack

UX/UI Strategist

Activated deeply when glanceability, accessibility, battery-aware rendering, or visible flow changes matter.

Role stack

Market Data Analyst

Activated deeply when data contracts, freshness guarantees, source choice, or polling cadence change.

4. Execution path
Primary route

Engineer

Implements only after premises are approved. Must not reopen product scope silently.

Primary route

QA

Validates acceptance, edge cases, regressions, and failure locality when validation is explicitly requested.

Primary route

Documentation / Historian

Consolidates the internal truth across docs and evidence so the repo remains coherent after delivery.

5. Runtime memory
Gate chain

Gate sequence

Understanding / Plan → Operational Specification → Implementation → Validation → Documentary Consolidation.

Persistence

Active request state

`lead_entrypoint.py` persists the active request, intake payload, and request log under `%LOCALAPPDATA%\\CodexLead\\Project Minos\\active`.

Persistence

Role report registry

`role_report_entrypoint.py` stores both the latest role report and a timestamped copy in `role_reports/` for traceability.

Recommendation router

Trivial patch Recommendation usually goes directly to Engineer with light validation.
Pure contract review Structural review without product or UX/data change routes to Documentation / Historian.
Behavior ambiguity Routes to Product Strategist before implementation starts.
Visible surface impact Routes to UX/UI Strategist when readability, glanceability, or interaction cost is material.
Data or freshness impact Routes to Market Data Analyst when source guarantees or polling logic are touched.

Depth model

Deep analysis Used for feature flows, structural work, or ambiguous requests that can drift without explicit premise work.
Light validation Used for bounded changes where a role mainly confirms no hidden drift or regression exists.
No-impact confirmation Used when a role is logically present in the stack but no meaningful impact is currently signaled.

Source mapping

  • `AGENTS.md` defines the intake contract, role stack, authority model, and persistence rule.
  • `docs/playbooks/activation_matrix.md` defines default role depth by change class.
  • `docs/agents/*.md` define each role mission, responsibilities, and prohibitions.
  • `tools/multiagent/lead_orchestrator.py` materializes classification, activation, gates, and recommendation logic.
  • `tools/multiagent/lead_entrypoint.py` materializes runtime persistence.
  • `tools/multiagent/role_report_entrypoint.py` materializes role report capture.
